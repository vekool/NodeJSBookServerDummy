üîê Authentication Endpoints

1. Login

Endpoint: POST /auth/login

Description: Authenticates a user and returns a JWT token. Accepts any username/password combination for testing purposes.

Authentication Required: No

Request Body

{
  "username": "string",
  "password": "string"
}

Success Response (200 OK)

{
  "message": "Login successful",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",

    "username": "admin"

{
  "error": "Username and password required"
}

cURL Example

curl -X POST http://localhost:3001/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin123"}'

üéì RxJS Teaching / WebSocket Streams

Real-time WebSocket streams for teaching RxJS operators. This server provides configurable data streams (books and issues) that emit at custom intervals with error injection, duplicates, and delays for comprehensive operator teaching.

WebSocket Connection

URL: ws://localhost:3001

Library: Socket.io (install in Angular: npm install socket.io-client)

Endpoint: GET /rxjs/streams

curl http://localhost:3001/rxjs/streams

Endpoint: POST /rxjs/streams/start

Description: Start a stream with custom configuration.

{
  "streamName": "books",
  "interval": 2000,
  "duration": 60000,
  "errorRate": 10,
  "duplicateRate": 0,
  "delayVariation": 500,
  "burstMode": false
}

3. Stop Stream

Endpoint: POST /rxjs/streams/stop/:streamName

Description: Stop a specific stream.

4. Stop All Streams

Endpoint: POST /rxjs/streams/stop-all

Description: Stop all running streams.

5. Get Preset Configurations

Endpoint: GET /rxjs/presets

Description: Get pre-configured scenarios for teaching specific operators.

Available Presets:

  • basic - Simple streams for map, filter, tap
  • throttleDebounce - Fast bursts for throttle vs debounce
  • errorHandling - Random errors for catchError, retry
  • distinctDuplicates - Duplicates for distinctUntilChanged
  • combination - Two streams for combineLatest, merge, zip
  • switching - Variable delays for switchMap, mergeMap
  • buffering - Fast stream for buffer operators
  • timing - Variable timing for delay, timeout

6. Start Preset

Endpoint: POST /rxjs/presets/:presetName

Example:

curl -X POST http://localhost:3001/rxjs/presets/combination

7. Get Operators Guide

Endpoint: GET /rxjs/operators-guide

Angular Integration Example

import { Injectable } from '@angular/core';
import { io, Socket } from 'socket.io-client';
import { Observable, fromEvent } from 'rxjs';
import { map } from 'rxjs/operators';

@Injectable({ providedIn: 'root' })
export class SocketService {
  private socket: Socket;

  constructor() {
    this.socket = io('http://localhost:3001');
  }

  // Convert socket events to Observables
  onBooks(): Observable<any> {
    return fromEvent(this.socket, 'books');
  }

  onIssues(): Observable<any> {
    return fromEvent(this.socket, 'issues');
  }

  // Emit events to server
  startStream(config: any): void {
    this.socket.emit('start-stream', config);
  }

  stopStream(name: string): void {
    this.socket.emit('stop-stream', name);
  }
}

Angular HttpClient Sample code to get you started

AuthService Implementation

import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';
import { tap } from 'rxjs/operators';

@Injectable({
  providedIn: 'root'
})
export class AuthService {
  private apiUrl = 'http://localhost:3001/auth';
  private tokenKey = 'auth_token';

  constructor(private http: HttpClient) {}

  login(username: string, password: string): Observable<any> {
    return this.http.post<any>(`${this.apiUrl}/login`, { username, password }).pipe(
      tap(response => {
        if (response.token) {
          localStorage.setItem(this.tokenKey, response.token);
        }
      })
    );
  }

  logout(): Observable<any> {
    return this.http.post(`${this.apiUrl}/logout`, {}).pipe(
      tap(() => {
        localStorage.removeItem(this.tokenKey);
      })
    );
  }

  getToken(): string | null {
    return localStorage.getItem(this.tokenKey);
  }

  isLoggedIn(): boolean {
    return !!this.getToken();
  }
}

üìù Notes

  • Mock Authentication: The server accepts ANY username/password combination for testing purposes.
  • In-Memory Storage: All data is stored in memory and resets when the server restarts.
  • CORS Enabled: The server has CORS enabled to allow requests from Angular dev server (http://localhost:4200).
  • Request Logging: All requests are logged to the console with headers for debugging.
  • Token Storage: Tokens are tracked server-side for logout functionality.
  • Network Delay: All endpoints simulate a 500ms network delay for realistic testing.
